# 设计文档：企微推送从 SCF 迁移到 Gitee Go

**日期：** 2026-06-21
**状态：** 已批准（修订版 v2）
**目标：** 停用腾讯云 SCF 扣费服务，将企微每日推送迁移到 Gitee Go（码云 CI），免费、国内可访问、关机也能跑。

---

## 背景

项目目前存在两套企微每日推送机制：

| 推送方式 | 状态 | 费用 | 国内可访问 |
|----------|------|------|:--:|
| SCF 云函数 | ✅ 正常运行 | ❌ 扣费 | ✅ |
| GitHub Actions | ❌ Secret 未配置 | ✅ 免费 | ❌ 被墙 |

最初计划迁移到 GitHub Actions，但发现 GitHub 在国内无法访问。项目代码已同步到 Gitee（`gitee/master`），Gitee 提供免费的 CI 服务「Gitee Go」，国内直连无压力。

**同时，SCF 可能并未真正超免费额度**——值得先确认 SCF 的实际消费明细（可能只是几毛钱的基础资源费，也可能确实超出了免费额度）。如果 SCF 费用可忽略，直接保留 SCF 也是合理选择。本方案默认走向迁移。

## 目标架构

```
迁移前：
  SCF 云函数 ──→ 企业微信群机器人
  扣费 ❌

迁移后：
  SCF 云函数（已停用）
  Gitee Go ──→ 企业微信群机器人
  免费 ✅  国内直连 ✅
```

**回滚通道：** 本地 `scripts/wechat_push.sh`（手头有电脑时手动触发）

## 执行步骤

### 第 1 步：确认 SCF 实际消费

登录腾讯云费用中心，查看 SCF 的具体扣费明细和金额。若费用极低（如每月几毛），可评估是否值得迁移。

### 第 2 步：创建 Gitee Go 工作流

在仓库中创建 `.gitee-ci.yml`，配置每日定时触发企微推送。

### 第 3 步：配置 Gitee 环境变量

在 Gitee 仓库设置中添加 `WECHAT_WEBHOOK_URL` 作为私密变量。

### 第 4 步：手动触发验证

手动触发 Gitee Go 流水线，确认企微群收到推送消息。

### 第 5 步：停用 SCF

确认 Gitee Go 推送正常后，登录腾讯云控制台停用 SCF 云函数。

### 第 6 步：观察验证

观察 2-3 天，确认每日定时推送正常到达。

## 涉及文件

| 文件 | 操作 |
|------|------|
| `.gitee-ci.yml` | **新建** — Gitee Go 工作流配置 |
| `.github/workflows/wechat-reminder.yml` | 保留，不再使用 |
| `scripts/wechat_push.sh` | 保留，回滚通道 |
| `scf/grant_perm.py` | 保留，标记废弃 |

## 回滚方案

如果 Gitee Go 推送出现问题：

```bash
# 本地手动推送应急
bash scripts/wechat_push.sh "⏰ 每日提醒\n\n今日运营数据请及时汇总，如有异常请及时反馈。"
```

**不再重新启用 SCF。**
